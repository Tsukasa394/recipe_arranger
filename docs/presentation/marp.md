# プレゼン資料作成方針

## 1. 資料作成方法

### 1.1 作成環境
- マークダウンのmarp記法を利用する
- 図表はmermaid記法を利用する（mermaid-cliでSVGに変換）
- marp-cliでPDFに出力する

### 1.2 準備物
- **marp.md**: プレゼン資料作成方針（本資料）
- **presentation.md**: プレゼン資料用マークダウン
- **style.css**: プレゼン資料デザインCSS
- **diagrams/**: mermaid図表のmmdファイル格納ディレクトリ
- **images/**: 変換後のSVG画像格納ディレクトリ

### 1.2.1 Mermaid図表の管理
- mermaid図表は `diagrams/*.mmd` ファイルとして独立して管理
- presentation.mdには直接mermaid表記を書かず、SVG画像として参照
- mermaid-cliを使用してmmdファイルをSVGに変換

### 1.3 環境変数（プレースホルダー）
- **AUTHOR**: 作成者の所属と氏名
- presentation.mdでは `{{AUTHOR}}` というプレースホルダーを使用する
- PDF出力前に、テキストエディタの検索置換機能などで `{{AUTHOR}}` を実際の値に置換すること
- 例: `{{AUTHOR}}` → `株式会社○○ 山田太郎`

### 1.4 Mermaid図表の変換
mermaid-cliを使用してmmdファイルをSVGに変換します。

**変換コマンド**:
```bash
cd docs/presentation/diagrams
./convert-to-svg.sh
```

**注意**: PDF出力前に、必ずmermaid図表をSVGに変換しておくこと

### 1.5 エクスポート
````
marp presentation.md -o presentation.pdf --allow-local-files
````

### 1.6 参照資料
- **[プロダクト概要](./docs/design/product.md)**: プロジェクトの背景、目的、機能仕様
- **[システム仕様書](./docs/design/spec.md)**: 詳細な機能要件、API仕様、デプロイ仕様

### 2.2 含めたい内容

#### 動機・背景
- せっかくだから生成AIを活用したアプリが作りたかった
- 現地参加でデモができないため、バックエンドが動く環境を公開したかった
- 開発手法として仕様駆動開発っぽいことをしたかった
- Spec Kitを使った仕様駆動開発にもチャレンジしたが使い方をしっかり学んでからチャレンジする必要があると感じた

#### 技術的な課題
- Gemini APIのJSライブラリが新しくなっているが、Cursorが最新ライブラリを認識してくれず大変だった

#### 開発手法の学び
- AIに任せすぎると暴走してプログラムを破壊する。細分化して適切に指示して伴走的に作業を見守ってあげる必要がある
- デバッグをAIに任せたが、テスト用ソースコードに間違えがあるのに、テスト結果が誤っていると勘違いしてソースコードに改悪を繰り返した

#### 今後の展望
- お気に入り機能の追加
- 残り物の写真を判断する機能

### 2.3 注意点
- **デモ**: VercelへのアクセスURLとスクリーンショットを使用する
  - スクリーンショットは主要な画面（トップページ、レシピ生成結果など）を1〜2枚程度
  - スクリーンショットの配置は `docs/presentation/images/` ディレクトリに保存し、相対パスで参照する
  - 推奨サイズ: 幅1200px程度（スライドに適切に表示されるサイズ）
- **ページ数**: 資料は全体で20ページ以内に収める
